# 25Maths System Glossary

> Canonical terminology for development, AI collaboration, and external communication.
> All contributors (human or AI) should use these terms consistently.

---

## Core Objects

### Learning Unit
The smallest trackable learning object in the system. Three types exist:
- **Vocabulary** — a word-definition pair (e.g. "integer: a whole number")
- **Knowledge Point (KP)** — a syllabus-level concept (e.g. "1.3 Squares, cubes, roots")
- **Question** — a past paper question or MCQ practice item

All three types are tracked by the same FLM state machine.

### Learning Graph
The runtime relationship layer that connects Learning Units across dimensions. Links vocabulary, knowledge points, and questions via shared section codes. Implemented in `learning-graph.js`.

### Recovery Session
A chained sequence of refresh scans launched from Today's Plan. Orchestrates vocab -> KP -> PP scans as a single task flow with step indicator, result panels, and Next/Exit controls. Implemented in `recovery-session.js`.

### Recovery Pack
A contextual repair bundle shown when a student rates a question as "needs work". Contains: weak vocabulary links, related KP links, similar questions, and a Print Repair Sheet button. Rendered inline in the question result card.

### Repair Sheet
A single-question A4 printable worksheet for paper-based correction. Contains: question text, related vocabulary, KP summary, blank working space, and self-correction section. Generated by `worksheet.js`, rendered with KaTeX in a new browser window.

### Section Health
A 0-100 composite score for a syllabus section, computed from four FLM-weighted sub-scores (vocab, practice, KP, past paper). Drives home page recommendations, Smart Path highlighting, and recovery prioritization.

## State & Tracking

### FLM (Filter-Learn-Master)
The 4-state mastery model used for all Learning Units: `new -> learning -> uncertain -> mastered`. Replaced the original SRS (Spaced Repetition System) in v2.6.

### Stale
A mastered item whose last review exceeds the expected refresh interval. Stale items enter the recovery queue. Detected by `getStaleWords()`, `getStaleKPs()`, `getStalePPQuestions()`.

### Refresh Scan
A three-button scan (Know / Fuzzy / Don't know) applied to stale items. Exists for all three Learning Unit types: vocab refresh, KP refresh, PP refresh.

### Mastered Decay
The mechanism by which mastered items lose their status over time. Governed by `REFRESH_INTERVALS = [7, 14, 30]` days and `rc` (refresh count, max 2).

### Reflow
The process of returning a mastered vocabulary word to the learning pool when a related question is answered incorrectly. Triggered by `reflowVocabForSection()`.

### cs (Consecutive Success)
The streak counter that drives FLM state transitions. Definition varies by dimension:
- Vocabulary: per-word correct streak
- KP: consecutive sessions with >= 85% accuracy
- PP: per-question correct streak (practice mode) or single high-confidence pass (exam mode)

### rc (Refresh Count)
How many times a mastered item has been successfully refreshed. Determines the next refresh interval. Capped at `MAX_RC = 2`.

## Planning & Execution

### Today's Plan
The daily planning dashboard that aggregates all pending learning tasks: stale items, weak sections, incomplete modes, and recovery recommendations.

### Weak Groups
Clusters of questions sharing the same command type (e.g. "Calculate", "Explain") where the student's accuracy is below threshold. Detected by `ppGetWeakGroups()`.

### Smart Path
The recommended learning sequence shown on the syllabus page. Highlights weak sections and suggests the next section to study based on Section Health scores.

### Recovery Priority (v3.5 planned)
A scoring function that ranks stale items by urgency: `priority = error_weight + decay_weight + exam_weight`. Will replace the fixed vocab -> KP -> PP ordering in Recovery Session.

## Content & Curriculum

### Board
An exam board or curriculum: CIE (0580), Edexcel (4MA1), or HHK (Harrow Haikou Y7-Y11).

### Section
A syllabus subdivision (e.g. "S2.5 Simultaneous equations"). The primary join key between vocabulary, KPs, and questions in the Learning Graph.

### Level
A vocabulary deck containing 5-20 word-definition pairs. 275 levels across all boards.

### Deck
The UI view of a Level — shows word list, FLM progress, mode buttons, and pool status.

### Pool
The set of non-mastered words in a level. Pool words cycle through scan rounds until all reach mastered status.

## Modes

### Scan (Study)
Three-button FLM cycle: Know / Fuzzy / Don't know. The primary vocabulary learning mode.

### Quiz
Four-choice MCQ mode for vocabulary reinforcement.

### Spell
Audio-driven spelling mode using Web Speech API.

### Match
Drag-and-drop pair matching for vocabulary.

### Battle
Timed matching challenge with scoring.

### Daily Challenge
A daily quiz drawing from all visible vocabulary, with streak tracking.

### Practice
MCQ practice using curated questions from the 25Maths website.

### Exam
Timed full-paper simulation using past paper data.

---

## Infrastructure

### Panel
A full-screen UI view managed by `showPanel(id)`. 18 panels total (home, deck, study, quiz, practice, etc.).

### Cloud Sync
Bidirectional sync between localStorage and Supabase `vocab_progress` table. Triggered on login, periodic save, and explicit sync.

### PWA
Progressive Web App with service worker (three-layer caching), manifest, install prompt, and offline detection.

### Build
esbuild minification: 16 core JS -> `app.bundle.min.js`, homework -> `homework.min.js` (lazy), CSS -> `style.min.css`. Run via `npm run build`.
